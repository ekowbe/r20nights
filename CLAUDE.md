# R20 Nights

**Read [HANDOFF.md](HANDOFF.md) first** — it's the living state of this project (what's
live, latest work, next steps).

## Handoff discipline
- Keep `HANDOFF.md` **lean (≤ ~50 lines)**: status, next steps, key commands, gotchas only.
- When you finish work, **update** "Current state" / "Next steps" in place — don't append
  a running log.

## Project rules
- Static site — plain HTML/CSS, **no build step**. Preview by opening `index.html`; deploy
  by pushing `main` (auto-deploys to Vercel via `github.com/ekowbe/r20nights`).
- Website code only. R20 *ministry* docs (leadership guide, meeting notes) stay on the
  Desktop, not in this repo.
